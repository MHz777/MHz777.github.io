MVC与MVVM

发表于 2020-04-28 | 分类于 Vue学习

MVC

简单来说，MVC是一种软件开发的思想，全名是Model View Controller，是模型(model)－视图(view)－控制器(controller)的缩写。
MVC的设计模式是将系统分为三个层：Model层，View层和Controller层。

Model层：简单来说就是进行数据的存储以及数据库的读取。

View层：视图层，简单说就是展示给用户的界面。

Controller层：控制业务逻辑，监听数据的改变和对用户的操作进行处理并做出相应。

MVVM

MVVM是Model-View-ViewModel的简写。它本质上就是MVC 的改进版。MVVM是一种前端分层的设计思路，
分为Model层，View层，ViewModel层。

Model层：控制数据的存储。

View层：展示给用户的界面，它的任务就是从ViewModel层获取数据然后展示。

ViewModel层：是View层和model层之间的桥梁，说白了就是将原来的ViewController层的业务逻辑和页面逻辑等剥离出来放到ViewModel层。

ThinkPHP实现like模糊查询

发表于 2020-03-22 | 分类于 ThinkPHP

本文讲述了如何使用thinkphp实现like模糊查询，分享给大家供大家参考

最近在做毕业设计，做一个小程序，在设计搜索栏的时候，发现如果不使用模糊查询的话，会使很多内容都检索不到，使用模糊查询的话，会使用户的体验更佳。

一、MySQL中模糊查询

MySQL的模式匹配允许使用“_”匹配任何单个字符，而“%”匹配任意数目字符（包括零个字符）。在 MySQL中，SQL的模式缺省是忽略大小写的。

%：表示任意个或多个字符。可匹配任意类型和长度的字符

比如 SELECT * FROM user WHERE user_name LIKE ‘%三%’

将会把user_name为“张三”，“张李三”、“三只猫”，“唐三”等等有“三”的记录全找出来。

二、数组方式作为查询条件

在ThinkPHP中like查询怎么实现呢，方法如下：

$User = M("User"); // 实例化User数据表
$where = array('like');//设置查询条件
$where['username'] = array('like','%张%');

$result = $User->where($where)->select();
print_r($result);//返回数据为查询结果

代码执行后会打印出查询结果，所有包含有“三”的字符串都会被检索出来。

小程序中全局变量的使用

全局变量可用于页面之间的跳转传参，在微信小程序中，页面的跳转方式有四种：

  • wx.navigateTO（保留当前页，跳转到应用内的其他页面，跳转非tabBar页面，可带参数）
  • wx.redirctTo（关闭当前页面，跳转到应用内的某个页面，跳转非tabBar页面，可带参数）
  • wx.switchTab（跳转到 tabBar 页面，并关闭其他所有非 tabBar 页面 不可带参数）
  • navigateBack（关闭当前页面，返回上一页面或多级页面）

我在设计一款小程序的时候发现，当页面跳转到tabbar页面的时候，并不能携带参数，所以为了解决这个问题，我研究了如何使用全局变量来让页面跳转到tab栏页面并传递参数。

话不多说直接上代码：

//app.js

App({

globalData: {
    userInfo: null,
    City:''
}
})


//job.js
var app = getApp();  //使用getAPP()方法获取实例
page({
    data:{
        city:'beijing'
    },

    submit:function (e) { 

    app.globalData.City = this.data.city;
    }
})


//index.js
var app = getApp();  //使用getAPP()方法获取实例
page({
    data:{
        city:''
    },
    onShow: function () {
    var city = app.globalData.City;
    console.log(city);

    this.setData({
        city:city
    })
    console.log(app.globalData);  //{userInfo: null, City: "beijing"}
    },
})

微信小程序中事件冒泡

发表于 2020-03-02 | 分类于 小程序

微信小程序中事件冒泡

什么是事件冒泡？

例如页面上的元素绑定了有很多事件：

<BODY onclick="alert('aaa');">
<div onclick="alert('bbb');">
 <a href="#" class="cooltip" title="这是我的超链接提示1。" onclick="alert('ddd');">
   提示
  </a>
</div>
</BODY>

当点击a标签的时候触发了点击事件，页面弹出提示框ddd，之后a标签的父元素div和body会依次触发各自绑定的事件，这样的现象就是事件冒泡。

微信小程序中的事件冒泡

在微信小程序中，事件的类别分为几种：

点击事件：tap

长按事件：longtap

触摸事件：touchstart； touchend；touchcancel；touchmove

其他：input；submit….

其中前三类是冒泡事件，其他的称为非冒泡事件。

如何阻止事件冒泡？

  1. 使用 catch 不用bind
    bind事件绑定不会阻止冒泡事件向上冒泡，catch事件绑定可以阻止冒泡事件向上冒泡。例：

    <view id="outer"          bindtap="handleTap1">
  outer view
  <view id="middle"          catchtap="handleTap2">
    middle view
    <view id="inner"          bindtap="handleTap3">
      inner view
    </view>
  </view>
</view>

  2. 在方法最后加上return false即可

AJAX - onreadystatechange 事件

onreadystatechange 事件

当请求被发送到服务器时，需要执行一些基于响应的任务。
每当 readyState 改变时，就会触发 onreadystatechange 事件。
readyState 属性存有 XMLHttpRequest 的状态信息。

下面是 XMLHttpRequest 对象的三个重要的属性：

属性 描述
onreadystatechange 存储函数（或函数名），每当 readyState 属性改变时，就会调用该函数。
readyState

存有 XMLHttpRequest 的状态。从 0 到 4 发生变化。

  • 0: 请求未初始化
  • 1: 服务器连接已建立
  • 2: 请求已接收
  • 3: 请求处理中
  • 4: 请求已完成，且响应已就绪
status 200: "服务器成功返回网页"
404: 未找到页面

代码验证：

//创建异步对象
var xhr = new XMLHttpRequest();

//设置请求行
xhr.open('get','./xxx.php');

//设置请求头
//xhr.setRequestHeader("Contant-type","x-www-form-urlencoded");

//注册回调函数
//xhr.onload
xhr.onreadystatechange=function(){
    //当xhr.readyState属性改变时,就会重新调用onreadystatechange函数

    console.log(xhr.status);

    if(xhr.status == 200 &&xhr.readyState==4){
        //判断当请求已完成，且响应已就绪返回内容
        console.log(xhr.readyState);//4
        console.log(xhr.status);//200
        console.log(xhr.responseText);//hello

    }


}

//发送请求主体 get请求为空，post请求写里面，没有不写
xhr.send();

三、学习Ajax

AJAX 工作原理

AJAX 实例

XMLHttpRequest 是 AJAX 的基础

<div id="box">
    <h1>哈喽我四jack dos，我四帅锅</h1>
    <input type="button" value="submit">
</div>

上面创建一个div，内容包含一个h标签和一个按钮。现在我们为这个按钮添加点击事件。

<script>
    window.onload = function () {
        document.querySelector('input').onclick = function () {
            //新建异步对象
            var xhr = new XMLHttpRequest();

            //设置请求行：发送方式，发送地址
            xhr.open('get', './getData.php?name=rose&skill=jump');

            //设置请求头
            //xhr.setRequestHeader();

            //设置回掉函数
            xhr.onload = function () {
                //console.log(xhr.responseText);
                document.querySelector('h1').innerHTML = xhr.responseText;
            }
            //请求主体 发送
            xhr.send(null);
        }
    }
</script>

上面js代码实现的功能是，用户点击按钮后，将get方式发送到服务器的数据，然后从相应的php页面获取数据，并将获取到的text文本数据设置到页面上的H标签中。
